# Technical Specification: ARINN Remediation & Stability Upgrade

## 1. Epistemic Stability (The Truth Verification Layer)

### Problem
The agent currently ingests information indiscriminately, leading to potential "knowledge poisoning" from low-quality sources.

### Solution Architecture
We introduce a **TruthVerificationLayer** module that acts as a middleware between Ingestion (Scribe/Scrapers) and Memory (Titan Graph).

#### A. Confidence Scoring Algorithm
Every fact node ($N_f$) is assigned a generic reliability score ($S_{rel}$) based on three weighted factors:
$$ S_{rel} = (W_s \cdot R_{source}) + (W_c \cdot V_{cross}) + (W_t \cdot D_{time}) $$

*   **Weighting ($W$)**: $W_s=0.4$ (Source), $W_c=0.4$ (Cross-Check), $W_t=0.2$ (Recency).
*   **Source Reputation ($R_{source}$)**:
    *   White-listed Domain (e.g., python.org, arxiv.org) = 1.0
    *   Git Commit (Verified Author) = 0.9
    *   General Web (HTTPS) = 0.5
    *   Unknown/HTTP = 0.1
*   **Cross-Validation ($V_{cross}$)**:
    *   Verified by `CrossVerifier` (found on >2 distinct domains) = 1.0
    *   Single Source = 0.5
    *   Contradicted = 0.0
*   **Time Decay ($D_{time}$)**:
    *   $\frac{1}{1 + \log(1 + \text{years\_old})}$

#### B. Provenance Schema (Graph Structure)
The Titan Graph (`networkx`) schema is strictly typed to enforce lineage.

```python
# Node Schema
{
    "id": "Fact_001",
    "content": "Python 3.13 was released in 2024",
    "meta": {
        "confidence": 0.92,
        "provenance": [
             {
                 "source_url": "https://docs.python.org/3/...",
                 "timestamp": "2025-01-20T10:00:00Z",
                 "hash": "sha256:..."
             }
        ]
    }
}
```

#### C. Garbage Collection (The Reaper)
A periodic process `MemoryReaper` scans the graph during idle cycles.
*   **Threshold**: Any node with $S_{rel} < 0.3$ is marked `DEPRECATED`.
*   **Pruning**: Deprecated nodes are archived to disk (cold storage) and removed from the active graph to prevent logic pollution.

## 2. Capability Transfer (Knowledge Distillation Pipeline)

### Problem
The system relies on the monolithic `NeuralCore` (TinyLlama/DistilGPT2) for all tasks, which is computationally expensive and creates a bottleneck.

### Solution Architecture
We implement a **Teacher-Student Distillation Loop**.

#### A. Curriculum Generation
The **Teacher** (TinyLlama) generates synthetic training data for a **Student** (e.g., a lightweight Scikit-Learn Classifier or a small LSTM `SubBrain`).

**Workflow**:
1.  **Topic selection**: Teacher picks a domain (e.g., "Is this code safe?").
2.  **Synthesis**: Teacher generates 50 examples of Safe Code and 50 examples of Malware, including a "Reasoning Trace".
    *   *Input*: `os.system('rm -rf /')`
    *   *Reasoning*: "Uses dangerous shell command deletion."
    *   *Label*: `UNSAFE`

#### B. Evaluation & Training Loop
```python
class DistillationEngine:
    def train_student(self, student_model, curriculum):
        for case in curriculum:
            prediction = student_model.predict(case['input'])
            
            # The Teacher critiques the Student
            if prediction != case['label']:
                # Feedback Loop
                student_model.adjust_weights(case, learning_rate=0.01)
                
    def evaluate(self, student_model, test_set):
        score = sum(1 for c in test_set if student_model.predict(c) == c.label)
        return score / len(test_set)
```

#### C. Independence Metric (Graduation)
A Student is "Graduated" (allowed to run in production without Teacher verification) when:
1.  **Accuracy**: >95% on a held-out Test Set generated by the Teacher.
2.  **Consistency**: Variance < 5% over 3 distinct evaluation runs.

## 3. Structural Coherence (The Constitutional Guardrail)

### Problem
Recursive self-improvement invites "Instrumental Convergence" where the agent might optimize away its own safety mechanisms to save compute.

### Solution Architecture
We enforce a **Constitution** backed by cryptographic immutability.

#### A. Immutable Core Invariants
A `manifest` defines files that are **Read-Only** to the Agent's own `Architect`.

```python
IMMUTABLE_CORE = {
    "sovereign.py": "sha256:abc...",   # The Main Loop
    "sandbox.py": "sha256:def...",     # The Safety Enforcer
    "constitution.py": "sha256:123..." # The Rules definitions
}
```
**Mechanism**:
Before `Architect.apply_change()` executes, it calculates the hash of the target file. If the file is in `IMMUTABLE_CORE`, modification is **Reject** immediately with a `ConstitutionalViolationError`.

#### B. Evolutionary Sandbox (Integration Testing)
Code changes are not just unit-tested; they are **simulated**.
1.  **Staging**: New code is deployed to `ARINN_Staging/`.
2.  **Drill**: The system spawns a "Ghost Process" using the new code.
3.  **Validation**: The Ghost Process must complete a standard "Calibration Task" (e.g., "Calculate 2+2", "Search Web") within standard time limits.
4.  **Merge**: Only if the Ghost Process exits successfully is the code promoted.

#### C. Semantic Drift Detection
To prevent goal drift (e.g., "Optimize Research" becoming "Delete Logs to reduce size"), we create a **Vector Anchor**.
1.  **Anchor**: At boot, we compute the embedding vector $V_{start}$ of the system's Prime Directive text using the Teacher LLM.
2.  **Check**: Every cycle, the agent describes its current plan $P$. We compute vector $V_{plan}$.
3.  **Distance**: If CosineSimilarity($V_{start}$, $V_{plan}$) < 0.7, the system triggers an **Emergency Halt** and requests Manual User Intervention.
